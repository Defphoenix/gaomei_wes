# v1.0 发布说明

- 版本：`v1.0.0`
- 发布日期：2026-07-17
- 定位：研发验证版

## 主要能力

1. 单样本胚系 HaplotypeCaller。
2. Tumor-normal paired Mutect2。
3. 污染估计、F1R2方向偏倚建模和FilterMutectCalls。
4. BQSR、比对后QC和BAM完整性校验。
5. VEP 115离线注释。
6. HLA*LA高分辨率G-group分型和class-I binding等位基因转换。
7. 基于Ensembl蛋白的8-15mer新抗原肽段和MHCflurry/NetMHCpan接口。
8. 严格VEP consequence TMB筛选。
9. CNVkit matched-normal、MSIsensor-pro、mosdepth覆盖度和中文汇总报告。
10. 一键完整运行以及 `step`、`from`、`through` 调试。

## v1.0 关键设计变更

配对项目不再分别对 normal BAM 和 tumor BAM 独立调用突变：

```text
normal FASTQ -> final BAM ----\
                               -> one paired Mutect2 -> somatic VCF
tumor FASTQ  -> final BAM ----/
```

这避免重复计算，也避免把 normal 的胚系 VCF 或 tumor-only VCF误当作配对体细胞结果。

## 真实数据复测推动的修复

- Picard 3.4使用 `MINIMUM_PCT` 兼容参数。
- samtools markdup不再传入旧版本不支持的 `--write-tags`。
- BQSR配置补齐known-sites变量，并在进入后续步骤前验证BAM。
- HLA*LA共享graph索引只构建一次。
- VEP明确传递`--dir_cache`并检查cache版本目录。
- Mutect2 `max-reads-per-alignment-start`恢复为50，避免损失低VAF证据。
- MSI解析器隔离原始结果和自己生成的summary，防止样本ID被误判为score。
- CNV无基线时不再生成伪CNR/CNS/VCF，只输出depth QC。
- mosdepth作为默认覆盖度工具，bedtools慢速全量统计默认关闭。

## 自动回归

`scripts/run_code_tests.sh` 覆盖：

1. Bash语法。
2. Python AST语法。
3. MSI旧输出递归解析回归。
4. HLA结果解析。
5. CNVkit matched-normal命令生成。

此外包含Mutect2辅助流程、严格TMB、新抗原和项目生成器模拟测试。

## 已知边界

- 缺少PoN和gnomAD AF-only时可研发运行，但假阳性控制不完整。
- 单matched normal CNV可复测，生产建议建立同平台pooled reference。
- MSI/TMB阈值必须按capture kit和癌种验证。
- Manta已归档，v1.0默认关闭SV。
- HLA*LA G-group结果不代表所有位点都无歧义达到8位。
- 新抗原binding预测不等于临床免疫原性结论。
- 当前HTML是研发报告，不是临床报告系统。
