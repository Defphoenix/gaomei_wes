## 2. 一致性规则

1. FASTA、捕获 BED、known-sites、Mutect2资源必须使用同一 GRCh38 build。
2. `chr1` 与 `1` 不能混用；参考字典顺序也必须一致。
3. VEP 115 必须使用 cache 115，推荐同时使用 Ensembl 115 protein FASTA。
4. Capture BED、MSI list、TMB denominator BED 和 CNV reference 应绑定到同一试剂盒版本。
5. PoN 应由同平台、同捕获试剂盒、同参考和同流程的 normal 构建。
6. 所有生产数据库应记录版本、下载日期、URL、文件大小和校验值。

## 4. 资源优先级

| 优先级 | 资源 | 用途 | 缺少时影响 |
|---:|---|---|---|
| P0 | GRCh38 FASTA及索引 | 全流程 | 无法比对或调用 |
| P0 | 捕获试剂盒BED | 调用、覆盖度、CNV、TMB | 区域和分母错误 |
| P0 | VEP 115 cache | 离线注释 | VEP不可用 |
| P0 | Ensembl 115 pep.all FASTA | 新抗原 | 无法恢复完整蛋白序列 |
| P1 | dbSNP/Mills/1000G known-sites | BQSR | BQSR跳过或不完整 |
| P1 | gnomAD AF-only | Mutect2 germline prior | 胚系先验不完整 |
| P1 | common SNP VCF | 污染估计 | contamination estimation跳过 |
| P1 | Panel of Normals | 批次伪影过滤 | 假阳性控制不足 |
| P1 | HLA*LA graph | HLA typing | 自动HLA分型不可用 |
| P1 | MHCflurry models | HLA-I binding | binding预测不可用 |
| P1 | CNVkit reference | 正式CNV | 只能matched normal或depth QC |
| P1 | MSI list/baseline | MSI | 不能形成验证结论 |
| P1 | effective coding BED | TMB分母 | 只能使用未验证分母 |

## 5. VEP、蛋白和新抗原

VEP cache 提供变异后果和转录本映射，但不是完整蛋白 FASTA 的替代品。新抗原模块
通过 VEP CSQ 中的 `Feature`/ENSP ID 到 Ensembl `pep.all` FASTA 查找原始蛋白，
再依据氨基酸改变构造突变蛋白和跨突变位点的 8-15mer 肽段。

推荐：

```text
VEP software: 115.x
VEP cache: homo_sapiens/115_GRCh38
Protein FASTA: Homo_sapiens.GRCh38.pep.all.fa from Ensembl release 115
```

如果三者版本错位，最常见结果是 ENSP 查找失败和新抗原数量异常减少。

## 6. 许可和不可自动下载资源

| 资源 | 原因 |
|---|---|
| NetMHCpan | DTU申请和许可 |
| ANNOVAR/humandb | 注册和许可 |
| COSMIC | 账户和许可 |
| 部分CADD/dbNSFP/REVEL | 许可、版本或体积限制 |

Conda 安装 VEP 不会自动获得这些数据，`--everything` 也不会自动提供所有插件评分。
