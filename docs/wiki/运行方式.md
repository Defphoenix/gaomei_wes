# 运行方式

## 1. 配对项目生成

```bash
PIPELINE_DIR=/PUBLIC/gomics/guofenghua/project/wes_pipeline
REFERENCE_DIR=/PUBLIC/gomics/guofenghua/reference_data
ENV_ROOT=/PUBLIC/gomics/guofenghua/envs/wes
OUT_DIR=/PUBLIC/gomics/guofenghua/project/wes_runs/TUMOR01_vs_NORMAL01

cd "${PIPELINE_DIR}"

bash scripts/create_wes_project.sh \
  --mode tumor-normal \
  --tumor-fastq-source /path/to/TUMOR01_fastq_dir \
  --normal-fastq-source /path/to/NORMAL01_fastq_dir \
  --out-dir "${OUT_DIR}" \
  --tumor-id TUMOR01 \
  --normal-id NORMAL01 \
  --project-name demo_project \
  --copy-mode link \
  --reference-dir "${REFERENCE_DIR}" \
  --reference-genome "${REFERENCE_DIR}/hg38/Homo_sapiens_assembly38.fasta" \
  --interval-bed "${REFERENCE_DIR}/capture_targets.bed" \
  --env-root "${ENV_ROOT}" \
  --no-testdata
```

项目生成器自动识别 R1/R2，建立软链接，复制流程代码并生成三份 config：

```text
configs/TUMOR01.align.config.sh
configs/NORMAL01.align.config.sh
configs/TUMOR01_vs_NORMAL01.somatic.config.sh
```

## 2. 一键完整运行

```bash
cd "${OUT_DIR}"
bash run_TUMOR01_vs_NORMAL01.sh
```

执行顺序是：

```text
normal through 5d -> tumor through 5d -> somatic from 6
```

Normal 和 tumor 不各自生成 VCF。只有 somatic 阶段将两份最终 BAM 一起输入 paired
Mutect2。

## 3. 分阶段运行

```bash
bash run_TUMOR01_vs_NORMAL01.sh normal
bash run_TUMOR01_vs_NORMAL01.sh tumor
bash run_TUMOR01_vs_NORMAL01.sh somatic
```

适合在服务器长任务中分别提交。只有 normal 和 tumor 的最终 BAM 都通过
`samtools quickcheck` 后才应开始 somatic。

## 4. 单步调试和恢复

项目 runner：

```bash
bash run_TUMOR01_vs_NORMAL01.sh step normal 5c
bash run_TUMOR01_vs_NORMAL01.sh through tumor 5d
bash run_TUMOR01_vs_NORMAL01.sh step somatic 7c
bash run_TUMOR01_vs_NORMAL01.sh from somatic 8
```

底层 runner：

```bash
bash pipeline/run_pipeline.sh \
  --config configs/TUMOR01_vs_NORMAL01.somatic.config.sh \
  step 7c
```

命令含义：

| 命令 | 作用 |
|---|---|
| `step X` | 只运行步骤X |
| `from X` | 从步骤X运行到结尾 |
| `through X` | 从步骤1运行到步骤X |
| `check` | 检查配置、软件、参考文件和输入 |

修复某一步后优先从失败步骤继续，不要删除已经验证完成的 BAM/VCF。

## 5. 单样本模式

单样本胚系模式使用 HaplotypeCaller，适用于没有配对 normal 的胚系分析。肿瘤单样本
Mutect2 可以作为研发模式运行，但无法获得 matched-normal 的胚系排除能力，不应与
标准 tumor-normal 结果等同解释。

## 6. 关键输出

```text
results/
  NORMAL01/bqsr/NORMAL01.bqsr.bam
  NORMAL01/hla_typing/
  TUMOR01/bqsr/TUMOR01.bqsr.bam
  TUMOR01_vs_NORMAL01/variants/*.mutect2.pass.vcf.gz
  TUMOR01_vs_NORMAL01/annotation/*.vep.vcf.gz
  TUMOR01_vs_NORMAL01/neoantigen/*.fa
  TUMOR01_vs_NORMAL01/neoantigen/*manifest.tsv
  TUMOR01_vs_NORMAL01/cnv/
  TUMOR01_vs_NORMAL01/msi/
  TUMOR01_vs_NORMAL01/tmb/
  TUMOR01_vs_NORMAL01/summary/
```

## 7. CNV和深度结果边界

- 有 matched normal 或 pooled reference：CNVkit 可输出 CNR/CNS/call。
- 没有任何CNV基线：mosdepth 只输出 `*.depth_qc.tsv`。
- `depth_qc.tsv` 不能解释成正式 CNV calls。

## 8. 运行前检查清单

1. R1/R2软链接存在且可读取。
2. FASTA、BED和VCF contig一致。
3. 主环境、VEP、HLA和CNV环境路径存在。
4. VEP cache目录包含 `homo_sapiens/115_GRCh38`。
5. protein FASTA可读且包含ENSP记录。
6. HLA graph和MHCflurry模型已准备。
7. 输出磁盘空间足够。
